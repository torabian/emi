Nullable object vs non-nullable

Emi definition allows for nullable object vs non-nullable objects. In case an object is not nullable, should always be present and initialised by the parent class upon instantiation. For types, simply it would indicate that it needs to be present.

name: nullableResponseAction
fields:
  - name: mother
    type: object
    description: Mother info should be present
    fields:
      - name: firstName
        type: string
  - name: father
    type: object?
    description: Father name is not essential for some goverment papers.
    fields:
      - name: firstName
        type: string
  - name: firstUncle
    type: one
    target: UncleDto
    description: Uncle is a separate dto, therefor we use that entity
  - name: secondUncle
    type: one?
    target: UncleDto
    description: Second uncle is optional
