The Kotlin compiler lets you generate type-safe clients and DTOs for Android and JVM projects out of the same Emi
definitions used by every other target. As with the other compilers, the generated Kotlin stays in sync with your
backend because it comes from one shared source of truth.

## Compiling a module

```bash
emi kotlin --path module.yaml --output ./src/main/kotlin/generated
```

Without `--output`, the generated virtual files are printed (as JSON) to the terminal so you can inspect them before
writing them to disk.

## Sub commands

<table>
  <thead>
    <tr>
      <th>Command</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>emi kotlin</code></td>
      <td>Compiles the entire Kotlin module (actions, DTOs, headers).</td>
    </tr>
    <tr>
      <td><code>emi kotlin:dto</code></td>
      <td>Generates only the Kotlin DTO objects.</td>
    </tr>
    <tr>
      <td><code>emi kotlin:headers</code></td>
      <td>Generates the headers, usable in both client and server code.</td>
    </tr>
  </tbody>
</table>

## Options

<table>
  <thead>
    <tr>
      <th>Flag</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>--path</code></td>
      <td>Path of the Emi definition file (<code>.yaml</code> or <code>.json</code>) on disk.</td>
    </tr>
    <tr>
      <td><code>--output</code></td>
      <td>Directory the generated Kotlin files are written to. Omit it to print to stdout.</td>
    </tr>
    <tr>
      <td><code>--tags</code></td>
      <td>Comma separated compile features to add or remove.</td>
    </tr>
  </tbody>
</table>

## Example

For the `postsModule` from the [OpenAPI](/emi/exports/openapi) page, `emi kotlin` generates a
`GetSinglePostAction.kt` for the action plus a shared `common.kt` containing the reusable building blocks (headers,
helpers) referenced by the actions.
